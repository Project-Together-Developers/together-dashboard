# i18n (Internationalization)

The application supports multiple languages using `react-i18next`.

## Setup and Locales

The configuration for i18n is initialized in `src/i18n/index.ts`.

- **Default Language**: `ru` (Russian)
- **Supported Languages**: `en`, `ru`, `uz`
- **Persistence**: The selected language is stored in `localStorage` under the key `together-admin-lang` to persist across sessions.

## Translation Files

Translation JSON files are located in `src/i18n/locales/`:
- `en.json`
- `ru.json`
- `uz.json`

**Important Rule**: Always add keys to *all three files* when adding new UI text.

## Usage

Use the `useTranslation` hook provided by `react-i18next` inside your components.

```typescript
import { useTranslation } from 'react-i18next';

export default function MyComponent() {
  const { t } = useTranslation();
  return <div>{t('namespace.key')}</div>;
}
```

## Error Codes Pattern

Use the enum pattern for backend error keys. An enum defines the keys that map directly to the i18n JSON files.

For example, `AuthErrorCode` (`src/utils/enums/auth-error-codes.ts`):
```typescript
export enum AuthErrorCode {
  INVALID_CREDENTIALS = 'auth.invalidCredentials',
  LOGIN_FAILED = 'auth.loginFailed',
}
```
Usage in code: `t(AuthErrorCode.INVALID_CREDENTIALS)`
